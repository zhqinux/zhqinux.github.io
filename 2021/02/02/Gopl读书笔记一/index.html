Gopl读书笔记(一)

Posted on 2021-02-02

这是英文版The Go Programming Language的读书笔记，很经典的一本书，可能是目前市面上最好的Go语言编程学习书籍，虽然有中译版，而且中译版的质量挺好，不过我还是想通读一遍英文版书籍，咀嚼别人的甘蔗渣，总不及自己咀嚼新的甘蔗来的香

又: Gopl这本书的第一章是几个Go语言编写的小程序，很多人看了第一章基本就劝退了（笑…），尤其是没有其他编程语言基础的，所以我还是建议希望学习Go的朋友先从Go官方Get Start

这个读书笔记是从第二章节开始的

2.1 Names

Go中函数、变量、常亮、类型、包等等元素的命名很简单

  • 以任意有效的unicode字符或下划线开始，后接任意数字或下划线或字符
  • 大小写敏感
  • 不能与Go保留的25个关键字重名

其实与其他编程语言没什么两样，只是关键字少了一些而已

2.2 Declarations

Go主要有四种声明，var(变量), const(常量), type(类型), func(函数)

在一个典型的Go文件中，首先是一个package语句，声明了这个文件属于哪个包

接下来是import语句，声明了依赖

然后是var,const声明，声明了包级别的变量和常量

最后就是各种函数声明

声明于函数内的var,const等等，属于local变量

声明于函数外的成为包级别变量

Go语言中有指针的概念，但要比C语言的指针简单多

一个指针的值，就是一个变量的地址，有时变量也被称为可寻址的值

如果大学学过C语言，那么理解Go的指针就相当简单了，Go的指针只能参与少数几个运算

比如解引用，相等性比较等等

另一种创建变量的方式是使用内置函数new

变量的生命周期指的是变量在程序执行期间存在的时间间隔

包级别的变量存在于整个程序的执行期

本地变量拥有着一个动态的生命周期，从它的声明语句开始，直到它unreachable（不可达）,也就意味着它的生命周期结束，等待垃圾回收机制自动回收它所占用的资源
